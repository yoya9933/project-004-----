# 2026-07-07 酌減比例迴歸模型建置紀錄

## 目標

依專題設計第四階段建立「酌減比例」模型管線。此階段先完成可重跑的比例目標表與回測腳本；正式金額標註不足時，腳本必須安全停止，不直接使用原始判賠金額作為目標，也不以 AI 金額候選替代人工確認金額。

## 使用資料

- 標註工作表：`06_交付物/ai_rag_annotation/annotation_workbook.csv`
- 待精標案件：120 件
- 主要目標：
  - `remaining_ratio = allowed_penalty / claimed_penalty`
  - `reduction_rate = 1 - remaining_ratio`
  - `reduction_bucket`
- 時間切分設定：2021-2024 訓練、2025 驗證、2026 測試

## 實作檔案

- `04_執行稿/run_reduction_ratio_model.ps1`

本腳本不依賴 Python、pandas、sklearn 或外部套件。現階段內建 mean baseline、Ridge Regression 與 Lasso Regression；Random Forest 與 LightGBM 留待 Python 套件環境與標註資料量穩定後再接入。

## 管線內容

- 讀取 `annotation_workbook.csv`。
- 產出 `ratio_model_frame.csv`，包含案件基本欄位、人工金額欄位、比例目標、分段目標、目標品質、模型特徵。
- 只使用正式 `claimed_penalty` 與 `allowed_penalty` 推導比例目標。
- `allowed_penalty` 只用於建立目標變數，不會放入模型特徵。
- 若 `remaining_ratio` 不在 0 到 1 之間，列為目標品質異常，不進入模型。
- 標籤達門檻後可輸出：
  - mean baseline
  - Ridge Regression
  - Lasso Regression
  - MAE、RMSE、R²、分段命中率
  - `metrics.csv`、`predictions.csv`、`model_coefficients.csv`

## 分段規則

| 分段 | 條件 |
|---|---|
| 未酌減 | `remaining_ratio >= 0.99` |
| 小幅酌減 | `0.70 <= remaining_ratio < 0.99` |
| 中度酌減 | `0.30 <= remaining_ratio < 0.70` |
| 大幅酌減 | `0.05 < remaining_ratio < 0.30` |
| 全免或近乎全免 | `remaining_ratio <= 0.05` |

## 注意事項

- 目前不能宣稱已完成酌減比例迴歸模型訓練，因為正式 `claimed_penalty` 與 `allowed_penalty` 仍為 0 筆。
- AI/RAG 產生的金額候選欄位可以輔助人工查核，但不能直接進入比例目標。
- 若同一判決同時有多個違約金項目，需人工判斷哪一個 `claimed_penalty` 與 `allowed_penalty` 是本模型要分析的同一請求項目，不能只取最大值或第一個金額。
- 下一步應優先補標 `claimed_penalty`、`allowed_penalty` 與 `contract_price`，再補 `delay_days` 與主要 issue 欄位，以支撐比例模型與後續分段分類。
