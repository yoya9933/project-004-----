## 先補哪些欄位

請優先在 `06_交付物/penalty_cases.csv` 補齊：

| 欄位 | 說明 |
|---|---|
| `contract_price` | 契約總價 |
| `delay_days` | 法院認定逾期天數 |
| `claimed_penalty` | 業主主張或扣罰違約金 |
| `allowed_penalty` | 法院最後准許違約金 |
| `key_reason` | 法院核心理由摘要 |
| `issue_actual_damage_unclear` | 業主實際損害不明 |
| `issue_partial_completion` | 工程已部分完成 |
| `issue_owner_fault` | 業主亦有責任 |
| `issue_contractor_fault` | 承包商可歸責 |
| `issue_extension_request` | 是否有展延申請 |
| `issue_used_by_owner` | 業主已使用工程成果 |

`amount_candidates_*` 欄位只是既有金額候選清單，用來輔助人工回判決找金額，不應直接當作契約總價或法院准許金額。

## 執行方式

PowerShell 中建議先設定 uv 目錄在 project004 內：

```powershell
$env:UV_CACHE_DIR=(Resolve-Path '.uv-cache').Path
$env:UV_PYTHON_INSTALL_DIR=(Resolve-Path '.uv-python').Path
```

重新生成主表骨架：

```powershell
uv run --with pandas --python 3.12 python "04_執行稿\build_penalty_cases_starter.py"
```

產生可交報告版：

```powershell
uv run --with pandas --with matplotlib --with tabulate --python 3.12 python "04_執行稿\analyze_penalty_cases_report.py"
```

產生進階線性模型版：

```powershell
uv run --with pandas --with scikit-learn --with tabulate --python 3.12 python "04_執行稿\run_penalty_linear_model.py"
```

## 報告建議說法

建議主軸：

> 本研究以工程逾期違約金案件為對象，將判決資料轉換為結構化欄位，觀察逾期天數、違約金占契約總價比例、法院准許金額與常見爭點因素之關聯。

模型限制：

> 線性模型僅作為探索性輔助分析，用於觀察本樣本中欄位與酌減結果的方向性關聯；因樣本數有限且判決事實差異大，不能作為預測法院判決或推論因果關係之依據。
