可以把實作分成 **「可交報告版」** 和 **「進階模型版」**。你目前已經有用 TPOT 針對 26 筆、11 個 0/1 爭點欄位做線性模型搜尋的方向；但這種樣本數很小，建議不要把它包裝成「預測法院」，而是包裝成 **判決資料標註＋探索性分析＋線性模型輔助解釋**。你貼的紀錄裡也有提到資料量是 26 筆、11 個爭點標籤，且 TPOT 嚴格線性版主要落在 LogisticRegression 與 LinearDiscriminantAnalysis 這類可解釋模型，這個方向適合作為輔助分析。

## 一、建議主軸

題目可以改得更工程資料分析一點：

> **工程逾期違約金酌減因素之判決資料分析：以逾期天數、違約金比例與法院酌減結果為中心**

不要主打「AI 預測判決」。
建議主打：

> **法院酌減與否，可能受到哪些可標註因素影響？**

因為法律上法院可依民法第252條酌減過高違約金；若有一部履行，也可能牽涉民法第251條的利益衡量。這些條文可以作為你們資料欄位設計的法律基礎。([法規資料庫][1])

## 三、幾個必要公式

假設：

* `contract_price` = 契約總價
* `claimed_penalty` = 原違約金
* `allowed_penalty` = 法院准許金額

公式如下：

```python
penalty_to_contract = claimed_penalty / contract_price
remaining_ratio = allowed_penalty / claimed_penalty
reduction_rate = 1 - remaining_ratio
is_reduced = 1 if allowed_penalty < claimed_penalty else 0
```

意義：

| 指標                    | 解釋         |
| --------------------- | ---------- |
| `penalty_to_contract` | 違約金占契約總價比例 |
| `remaining_ratio`     | 法院保留多少違約金  |
| `reduction_rate`      | 法院砍掉多少比例   |
| `is_reduced`          | 有沒有酌減      |

例如：

| 案件 | 原違約金 | 法院准許 | remaining_ratio | reduction_rate |
| -- | ---: | ---: | --------------: | -------------: |
| A案 | 100萬 | 100萬 |            100% |             0% |
| B案 | 100萬 |  70萬 |             70% |            30% |
| C案 | 100萬 |  40萬 |             40% |            60% |
| D案 | 100萬 |  25萬 |             25% |            75% |

## 五、Python 實作範例

### 1. 讀取資料與計算欄位

```python
import pandas as pd

df = pd.read_csv("penalty_cases.csv")

df["penalty_to_contract"] = df["claimed_penalty"] / df["contract_price"]
df["remaining_ratio"] = df["allowed_penalty"] / df["claimed_penalty"]
df["reduction_rate"] = 1 - df["remaining_ratio"]
df["is_reduced"] = (df["allowed_penalty"] < df["claimed_penalty"]).astype(int)

df.to_csv("penalty_cases_cleaned.csv", index=False)
```

### 2. 畫「逾期天數 × 酌減後比例」散佈圖

```python
import pandas as pd
import matplotlib.pyplot as plt

df = pd.read_csv("penalty_cases_cleaned.csv")

plt.figure(figsize=(8, 5))
plt.scatter(
    df["delay_days"],
    df["remaining_ratio"] * 100,
    s=(df["penalty_to_contract"] * 1000).clip(lower=30, upper=400),
    alpha=0.7
)

for _, row in df.iterrows():
    plt.text(
        row["delay_days"],
        row["remaining_ratio"] * 100,
        row["case_id"],
        fontsize=8
    )

plt.xlabel("逾期天數")
plt.ylabel("酌減後比例（%）")
plt.title("逾期天數與違約金酌減後比例")

plt.show()
```

這張圖的設定：

| 圖形元素 | 代表意義      |
| ---- | --------- |
| X 軸  | 逾期天數      |
| Y 軸  | 酌減後比例     |
| 點的大小 | 原違約金／契約總價 |
| 案件標籤 | case_id   |

## 六、建議加一張「分組統計表」

比散佈圖更容易寫結論。

```python
bins = [0, 0.05, 0.10, 0.20, 1.00]
labels = ["0-5%", "5-10%", "10-20%", "20%以上"]

df["penalty_ratio_group"] = pd.cut(
    df["penalty_to_contract"],
    bins=bins,
    labels=labels,
    include_lowest=True
)

summary = df.groupby("penalty_ratio_group").agg(
    cases=("case_id", "count"),
    reduced_cases=("is_reduced", "sum"),
    reduction_rate=("is_reduced", "mean"),
    avg_remaining_ratio=("remaining_ratio", "mean")
).reset_index()

summary["reduction_rate"] = summary["reduction_rate"] * 100
summary["avg_remaining_ratio"] = summary["avg_remaining_ratio"] * 100

print(summary)
summary.to_csv("penalty_ratio_group_summary.csv", index=False)
```

輸出會長這樣：

| 違約金占契約總價 | 案件數 | 酌減件數 |  酌減率 | 平均酌減後比例 |
| -------- | --: | ---: | ---: | ------: |
| 0-5%     |   5 |    1 |  20% |     90% |
| 5-10%    |   6 |    3 |  50% |     75% |
| 10-20%   |   5 |    4 |  80% |     55% |
| 20%以上    |   4 |    4 | 100% |     35% |

報告可以寫：

> 樣本中，原違約金占契約總價比例越高，法院酌減比例似有上升趨勢。但因樣本數有限，且法院仍會綜合考量實際損害、部分履行、業主是否有責、遲延原因等因素，因此本研究僅能作為探索性觀察，不能推論為固定規則。

## 八、建議模型實作：不要只靠 TPOT

TPOT 可以用來說明「自動搜尋模型」，但正式報告最好另外保留一個人工指定的 Logistic Regression，因為它比較好解釋。

```python
import pandas as pd
from sklearn.linear_model import LogisticRegression
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
from sklearn.model_selection import LeaveOneOut, cross_val_score
from sklearn.metrics import balanced_accuracy_score, make_scorer

df = pd.read_csv("penalty_cases_cleaned.csv")

features = [
    "delay_days",
    "penalty_to_contract",
    "issue_actual_damage_unclear",
    "issue_partial_completion",
    "issue_owner_fault",
    "issue_extension_request",
    "issue_used_by_owner",
]

X = df[features]
y = df["is_reduced"]

model = Pipeline([
    ("scaler", StandardScaler()),
    ("clf", LogisticRegression(max_iter=1000, class_weight="balanced"))
])

cv = LeaveOneOut()
scores = cross_val_score(
    model,
    X,
    y,
    cv=cv,
    scoring=make_scorer(balanced_accuracy_score)
)

print("平均 balanced accuracy:", scores.mean())

model.fit(X, y)

coef = pd.DataFrame({
    "feature": features,
    "coefficient": model.named_steps["clf"].coef_[0]
}).sort_values("coefficient", ascending=False)

print(coef)
coef.to_csv("logistic_regression_coefficients.csv", index=False)
```

係數解讀方式：

| 係數方向 | 意義                   |
| ---- | -------------------- |
| 正係數  | 該因素出現時，較傾向與「法院酌減」同向  |
| 負係數  | 該因素出現時，較傾向與「法院不酌減」同向 |
| 接近 0 | 關聯不明顯                |

但報告一定要加限制：

> 係數僅代表本樣本中的統計關聯，不代表法院實際法律判斷的因果關係。

## 十、報告架構可以這樣寫

### 1. 研究目的

分析工程逾期違約金案件中，法院酌減違約金時常見的判斷因素，並以資料標註與線性模型輔助觀察因素關聯。

### 2. 資料蒐集

蒐集臺灣工程契約逾期違約金判決，排除非工程契約、非逾期違約金、判決未討論違約金酌減、金額或天數不明確者。

### 3. 資料標註

將判決轉換為結構化欄位，包括逾期天數、契約總價、原違約金、法院准許金額、是否酌減、酌減理由與爭點特徵。

### 4. 分析方法

* 描述統計
* 散佈圖分析
* 違約金比例分組
* Logistic Regression 輔助分析
* AI 標註結果人工查核

### 5. 研究發現

可以分成：

* 違約金比例與酌減結果
* 逾期天數與酌減幅度
* 常見酌減理由
* 線性模型係數觀察
* AI 標註錯誤類型

### 6. 研究限制

一定要寫：

* 樣本數有限。
* 判決事實差異大。
* 法院判斷不是單一數字決定。
* AI 萃取可能誤判金額、逾期天數或判決結果。
* 模型只能輔助觀察，不能預測個案結果。

## 十一、最實際的分工方式

如果是 3～4 人小組，可以這樣分：

| 角色 | 工作                |
| -- | ----------------- |
| A  | 搜尋與篩選判決           |
| B  | 標註金額、天數、是否酌減      |
| C  | 標註爭點欄位與法院理由       |
| D  | Python 分析、畫圖、整理報告 |

如果人少，優先順序是：

1. 先完成 15 件判決標註。
2. 先畫散佈圖。
3. 再做分組統計。
4. 最後才做 TPOT 或 Logistic Regression。

不要一開始就卡在模型。這個題目的核心價值是 **判決資料結構化**，模型只是輔助。
